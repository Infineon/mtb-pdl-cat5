Field Documentation

struct BTSS_DMAC_APP_REQUEST* BTSS_DMAC_APP_REQUEST_t::next

Pointer to a pending transfer request on a shared channel.

The list is maintained by the DMA driver, and deferred requests will be initiated in first-in-first-out order whenever a previous transfer request has completed.

UINT8 BTSS_DMAC_APP_REQUEST_t::channel

DMA channel number on which the transfer is to be executed.

Which channel should be used is determined by a chip-specific DMA channel map, with constants named DMA_CHANNEL_*.

BTSS_DMAC_CALLBACK_t BTSS_DMAC_APP_REQUEST_t::callback

Function pointer specifying a function to be called when the DMA transfer has completed.

As the name indicates, this function is always called from the DMA complete interrupt. The function pointer can be NULL if there is some other mechanism for completing the transfer, though using callback is the safest way to assure that the DMA driver's state remains consistent with the state of the firmware that is using it.

void* BTSS_DMAC_APP_REQUEST_t::callback_arg

Arbitrary argument to be passed to callback_arg.

It can be used by that function to identify what transfer was completed, using whatever information seems appropriate to the designer. The argument could be a pointer to this very DMA_TRANSFER_REQUEST_t, but typically it is more useful to pass a pointer to the "larger" relevant state information block that described whatever the transfer was serving.

BTSS_DMAC_LLI_DESC_t BTSS_DMAC_APP_REQUEST_t::DMACCx_first_segment_regs

The register values that should be set in the DMA controller for the target channel.

These values will be written to the DMA controller's channel register bank when this transfer starts. DMA_LLI_t contains all those registers, although linked list items won't actually be used unless DMACCx_first_segment_regs.DMACCxLLI points to an actual linked list item.

UINT32 BTSS_DMAC_APP_REQUEST_t::DMACCxConfiguration
